# 风险管理实施指南

## 概述

风险管理是投资成功的关键。本指南介绍如何在投资计划中实施有效的风险管理，包括风险识别、评估、控制和监控。

## 风险识别

### 风险分类

#### 市场风险
- **系统性风险**: 整个市场下跌的风险
- **行业风险**: 特定行业面临的风险
- **风格风险**: 价值/成长风格切换风险
- **波动率风险**: 市场波动加剧的风险

#### 信用风险
- **违约风险**: 债券或交易对手违约
- **降级风险**: 信用评级下调
- **流动性风险**: 无法及时变现

#### 操作风险
- **执行风险**: 交易执行错误
- **系统风险**: 技术系统故障
- **人为风险**: 操作失误

#### 合规风险
- **监管风险**: 监管政策变化
- **法律风险**: 法律诉讼
- **税务风险**: 税务政策变化

### 风险识别矩阵

| 风险类型 | 风险描述 | 发生概率 | 影响程度 | 风险等级 |
|---------|---------|---------|---------|---------|
| 市场风险 | 市场整体下跌20% | 中 | 高 | 高 |
| 行业风险 | 行业政策收紧 | 低 | 高 | 中 |
| 公司风险 | 业绩不及预期 | 中 | 高 | 高 |
| 流动性风险 | 无法及时成交 | 低 | 中 | 低 |
| 操作风险 | 交易执行错误 | 低 | 低 | 低 |

## 风险评估

### 风险量化指标

#### 风险价值 (VaR)
```
VaR_95 = μ - 1.645 × σ
VaR_99 = μ - 2.326 × σ

解释:
- 95% VaR: 有95%的把握损失不会超过该值
- 99% VaR: 有99%的把握损失不会超过该值
```

#### 条件风险价值 (CVaR)
```
CVaR_95 = E[X | X ≤ VaR_95]

解释: 在最坏的5%情况下，平均损失是多少
```

#### 最大回撤
```
最大回撤 = (峰值 - 谷值) / 峰值

解释: 从最高点到最低点的最大跌幅
```

### 风险评分

| 指标 | 低风险 | 中风险 | 高风险 |
|------|--------|--------|--------|
| VaR_95 | < 5% | 5-10% | > 10% |
| CVaR_95 | < 8% | 8-15% | > 15% |
| 最大回撤 | < 10% | 10-20% | > 20% |
| 波动率 | < 15% | 15-30% | > 30% |

## 风险控制

### 事前控制

#### 仓位限制
```
单标的限制:
- 保守型: ≤ 10%
- 中性型: ≤ 20%
- 激进型: ≤ 30%

行业限制:
- 保守型: ≤ 25%
- 中性型: ≤ 40%
- 激进型: ≤ 60%

总仓位限制:
- 保守型: ≤ 60%
- 中性型: ≤ 80%
- 激进型: ≤ 100% (可杠杆)
```

#### 风险预算
```
单笔交易风险:
- 保守型: ≤ 1%
- 中性型: ≤ 2%
- 激进型: ≤ 3%

单日风险:
- 保守型: ≤ 3%
- 中性型: ≤ 5%
- 激进型: ≤ 8%

总体风险:
- 保守型: ≤ 10%
- 中性型: ≤ 15%
- 激进型: ≤ 25%
```

#### 止损设置
```
硬止损:
- 保守型: -5%至-8%
- 中性型: -8%至-12%
- 激进型: -12%至-15%

时间止损:
- 短线: 5-10天
- 中线: 1-3个月
- 长线: 6-12个月
```

### 事中控制

#### 实时监控
```python
class RiskMonitor:
    def __init__(self, portfolio):
        self.portfolio = portfolio
        self.alerts = []
    
    def monitor(self):
        # 监控价格
        self.check_price_limits()
        
        # 监控风险指标
        self.check_risk_metrics()
        
        # 监控仓位
        self.check_position_limits()
        
        # 监控相关性
        self.check_correlation()
        
        return self.alerts
    
    def check_price_limits(self):
        for position in self.portfolio.positions:
            if position.current_price <= position.stop_loss:
                self.alerts.append({
                    'type': 'stop_loss',
                    'symbol': position.symbol,
                    'message': f'{position.symbol} 触发止损'
                })
```

## 应急预案

### 市场极端情况

#### 闪崩应对
```
触发条件:
- 单标的日内跌幅 > 15%
- 大盘日内跌幅 > 10%

应对措施:
1. 暂停新增交易
2. 评估持仓风险
3. 检查止损有效性
4. 考虑对冲保护
5. 等待市场稳定
```

#### 流动性危机应对
```
触发条件:
- 买卖价差扩大 > 5%
- 成交量萎缩 > 50%

应对措施:
1. 降低仓位
2. 转向流动性好的标的
3. 增加现金比例
4. 避免强制平仓
```

#### 黑天鹅事件应对
```
触发条件:
- 重大突发事件
- 市场恐慌性抛售

应对措施:
1. 启动应急止损
2. 全面风险评估
3. 调整风险预算
4. 寻求对冲工具
5. 保持冷静，避免恐慌
```

### 操作风险应对

#### 交易错误应对
```
错误类型:
- 方向错误
- 数量错误
- 价格错误

应对措施:
1. 立即纠正
2. 评估损失
3. 记录原因
4. 完善流程
```

#### 系统故障应对
```
故障类型:
- 交易系统故障
- 网络中断
- 数据错误

应对措施:
1. 启用备用系统
2. 电话委托
3. 记录故障
4. 事后补偿
```
