# 投资分析定时任务稳定执行方案

## 问题诊断

### 失败原因分析

| 失败类型 | 频率 | 原因 | 解决方案 |
|---------|------|------|---------|
| AI服务过载 | 高 | 429 rate limit | 重试机制 + 错峰执行 |
| 工具验证错误 | 中 | Skill调用格式问题 | 简化任务指令 |
| 执行超时 | 中 | 任务运行时间过长 | 增加超时时间 |
| GitHub推送失败 | 低 | 网络问题 | 异步推送 + 本地备份 |

## 方案四：监控告警

### 执行状态监控

1. **心跳检测** - 每小时检查任务执行状态
2. **失败告警** - 连续2次失败发送通知
3. **成功确认** - 任务完成后确认GitHub推送

### 监控脚本

```bash
#!/bin/bash
# 检查最近任务执行状态

LOG_DIR="/root/.openclaw/workspace/logs"
STATUS_FILE="$LOG_DIR/last_run_status.json"

if [ -f "$STATUS_FILE" ]; then
    STATUS=$(cat "$STATUS_FILE" | grep -o '"status": "[^"]*"' | cut -d'"' -f4)
    if [ "$STATUS" = "failed" ]; then
        echo "⚠️ 上次任务执行失败，需要手动检查"
        # 发送通知
    fi
fi
```
