# 投资计划专家 (Investment Plan)

## 概述

本Skill是一个综合投资计划生成框架，整合**投资决策分析**、**投资交易执行**、**风险决策分析**三大模块的分析结论，生成完整、可执行的投资计划提案。

**核心理念**：
1. **系统化决策**: 基于多维度分析结果，系统化生成投资计划
2. **可执行性**: 提供具体、可操作的投资建议和交易方案
3. **风险控制**: 将风险管理嵌入投资计划的每个环节
4. **动态调整**: 根据市场变化和投资表现动态优化计划

## 投资计划框架

### 1. 投资主题 (Investment Thesis)

#### 核心投资逻辑
```
投资主题 = 宏观环境 + 行业趋势 + 公司优势 + 催化剂事件

示例:
"在AI浪潮推动下，云计算行业迎来爆发式增长。
该公司作为行业龙头，拥有技术壁垒和客户粘性，
近期发布的超预期财报成为股价上涨的催化剂。"
```

#### 关键假设
| 假设类别 | 具体假设 | 验证方法 | 风险点 |
|---------|---------|---------|--------|
| 宏观假设 | 经济软着陆 | GDP数据、就业数据 | 衰退风险 |
| 行业假设 | 行业增长20% | 行业报告、订单数据 | 增长不及预期 |
| 公司假设 | 市场份额提升 | 财报数据、调研 | 竞争加剧 |
| 估值假设 | PE维持25倍 | 同业比较 | 估值压缩 |

#### 催化剂事件
- **短期催化剂** (1-3个月)
  - 季度财报发布
  - 新产品上市
  - 重大合同签订
  
- **中期催化剂** (3-12个月)
  - 产能扩张完成
  - 新市场开拓
  - 行业政策出台
  
- **长期催化剂** (1-3年)
  - 技术突破
  - 行业整合
  - 国际化扩张

### 2. 交易策略 (Trading Strategy)

#### 入场策略

##### 入场条件清单
- [ ] 投资决策综合评分 ≥ 70分
- [ ] 技术形态符合要求
- [ ] 市场环境适宜
- [ ] 风险收益比 ≥ 1:3
- [ ] 流动性充足

##### 入场价格区间
```
理想入场价: [价格区间下限]
可接受入场价: [价格区间]
最高入场价: [价格区间上限]

入场价格确定方法:
1. 技术分析: 支撑位、均线位置
2. 估值分析: 合理估值区间
3. 市场情绪: 恐慌/贪婪指数
```

##### 入场时机选择
| 时机类型 | 适用场景 | 操作方法 | 风险等级 |
|---------|---------|---------|---------|
| 突破入场 | 技术形态突破 | 突破确认后买入 | 中 |
| 回调入场 | 上涨趋势中的回调 | 回调至支撑位买入 | 中低 |
| 分批入场 | 不确定性较高 | 分3-5批建仓 | 低 |
| 事件驱动 | 催化剂事件 | 事件确认后买入 | 高 |

#### 仓位管理策略

##### 初始仓位计算
```
风险比例法:
初始仓位 = 可承受亏损金额 ÷ (入场价 - 止损价)

固定比例法:
初始仓位 = 账户总资产 × 固定比例(如5%)

凯利公式:
f* = (bp - q) / b
其中: b=赔率, p=胜率, q=败率
```

##### 加仓策略

**金字塔加仓法** (推荐)
| 加仓次数 | 仓位比例 | 触发条件 | 累计仓位 |
|---------|---------|---------|---------|
| 首次 | 30% | 达到入场条件 | 30% |
| 第二次 | 20% | 盈利10%后 | 50% |
| 第三次 | 15% | 盈利20%后 | 65% |
| 第四次 | 10% | 盈利30%后 | 75% |

**倒金字塔加仓法** (谨慎使用)
| 加仓次数 | 仓位比例 | 触发条件 | 累计仓位 |
|---------|---------|---------|---------|
| 首次 | 10% | 达到入场条件 | 10% |
| 第二次 | 15% | 下跌5%后 | 25% |
| 第三次 | 20% | 下跌10%后 | 45% |
| 第四次 | 30% | 下跌15%后 | 75% |

##### 减仓策略

**止盈减仓**
| 盈利水平 | 减仓比例 | 剩余仓位 | 操作理由 |
|---------|---------|---------|---------|
| +15% | 减仓25% | 75% | 锁定部分利润 |
| +25% | 减仓25% | 50% | 回收本金 |
| +35% | 减仓25% | 25% | 保留利润头寸 |
| +50% | 清仓 | 0% | 目标达成 |

**风险减仓**
| 风险信号 | 减仓比例 | 操作理由 |
|---------|---------|---------|
| 基本面恶化 | 减仓50% | 降低风险敞口 |
| 技术破位 | 减仓30% | 趋势可能反转 |
| 宏观风险 | 减仓30% | 系统性风险 |
| 达到止损 | 清仓 | 控制损失 |

#### 出场策略

##### 止盈策略

**固定比例止盈**
- 目标收益: +25%、+35%、+50%
- 分批止盈: 达到目标后分批减仓
- 优点: 简单易执行
- 缺点: 可能错失更大涨幅

**移动止盈**
```
盈利20%: 止损上移至成本价 (保本)
盈利30%: 止损上移至盈利10%位置
盈利50%: 止损上移至盈利25%位置
盈利100%: 止损上移至盈利50%位置
```

**技术止盈**
- 到达重要阻力位
- 技术指标超买 (RSI > 80)
- 成交量异常放大
- 形态完成 (如双顶)

##### 止损策略

**固定比例止损**
- 硬止损: -7%、-10%、-15% (根据风险偏好)
- 优点: 简单明确
- 缺点: 可能被短期波动触发

**技术止损**
- 跌破关键支撑位
- 趋势线破位
- 形态失败 (如突破失败)
- 均线死叉

**时间止损**
- 持仓超过预期时间未达目标
- 避免资金占用成本
- 机会成本考虑

**波动率止损**
```
止损价 = 入场价 - 2 × ATR(14)

ATR(14): 14日平均真实波幅
```

### 3. 风险管理方案 (Risk Management)

#### 风险识别矩阵

| 风险类型 | 风险描述 | 发生概率 | 影响程度 | 风险等级 |
|---------|---------|---------|---------|---------|
| 市场风险 | 市场整体下跌 | 中 | 高 | 高 |
| 行业风险 | 行业政策变化 | 低 | 高 | 中 |
| 公司风险 | 业绩不及预期 | 中 | 高 | 高 |
| 流动性风险 | 无法及时成交 | 低 | 中 | 低 |
| 操作风险 | 执行错误 | 低 | 低 | 低 |

#### 风险控制措施

##### 事前控制
- **仓位限制**: 单标的不超过账户20%
- **行业分散**: 单一行业不超过账户40%
- **止损设置**: 每笔交易必须设置止损
- **风险预算**: 单笔交易风险不超过账户2%

##### 事中控制
- **实时监控**: 监控价格、波动率、新闻
- **预警机制**: 接近止损位时预警
- **情绪管理**: 避免情绪化操作
- **执行纪律**: 严格执行交易计划

##### 事后控制
- **交易日志**: 记录每笔交易
- **定期复盘**: 分析交易结果
- **策略优化**: 根据反馈优化策略
- **风险回顾**: 评估风险管理效果

#### 应急预案

##### 市场极端情况应对
**闪崩应对**
- 暂停新增交易
- 评估持仓风险
- 考虑对冲保护
- 等待市场稳定

**流动性危机应对**
- 降低仓位
- 转向流动性好的标的
- 增加现金比例
- 避免强制平仓

**黑天鹅事件应对**
- 启动应急止损
- 全面风险评估
- 调整风险预算
- 寻求对冲工具

### 4. 执行方案 (Execution Plan)

#### 执行时间表

| 阶段 | 时间 | 任务 | 负责人 | 交付物 |
|------|------|------|--------|--------|
| 准备阶段 | T-7天 | 完成分析、制定计划 | 分析师 | 投资计划书 |
| 审批阶段 | T-3天 | 计划审批、资金准备 | 投资经理 | 审批文件 |
| 建仓阶段 | T日 | 首次建仓 | 交易员 | 成交确认 |
| 监控阶段 | T+1起 | 持续监控、动态调整 | 风控 | 监控报告 |
| 退出阶段 | 触发条件 | 止盈/止损出场 | 交易员 | 结算报告 |

#### 执行检查清单

**建仓前检查**
- [ ] 投资计划已审批
- [ ] 资金已到位
- [ ] 交易权限已确认
- [ ] 风险限额已设定
- [ ] 止损单已准备

**建仓时检查**
- [ ] 价格符合计划区间
- [ ] 流动性充足
- [ ] 仓位大小符合计划
- [ ] 止损单已设置
- [ ] 交易记录已保存

**持仓期检查**
- [ ] 每日监控价格
- [ ] 关注新闻事件
- [ ] 评估风险状况
- [ ] 检查止损有效性
- [ ] 记录交易日志

**出场时检查**
- [ ] 出场条件已触发
- [ ] 出场价格确认
- [ ] 收益/损失计算
- [ ] 交易记录更新
- [ ] 复盘总结完成

### 5. 监控与调整 (Monitoring & Adjustment)

#### 关键监控指标

##### 价格指标
- 当前价格 vs 入场价
- 当前价格 vs 止损价
- 当前价格 vs 目标价
- 价格波动率

##### 风险指标
- 当前回撤
- 风险收益比变化
- 仓位占比
- 组合相关性

##### 市场指标
- 大盘走势
- 行业表现
- 资金流向
- 情绪指标

#### 调整触发条件

| 触发条件 | 调整措施 | 优先级 |
|---------|---------|--------|
| 达到止损位 | 立即止损 | 紧急 |
| 基本面恶化 | 减仓或清仓 | 高 |
| 达到第一止盈 | 减仓25% | 中 |
| 波动率异常 | 评估调整止损 | 中 |
| 市场环境变化 | 重新评估计划 | 中 |
| 达到时间止损 | 评估退出 | 低 |

#### 定期回顾机制

**每日回顾**
- 检查持仓状态
- 监控风险指标
- 关注市场新闻

**每周回顾**
- 评估交易表现
- 分析盈亏原因
- 调整风险参数

**每月回顾**
- 全面绩效评估
- 策略有效性分析
- 计划优化更新

## 使用方法

### 生成完整投资计划

```bash
python scripts/generate_investment_plan.py \
  --symbol AAPL \
  --decision-file ./output/AAPL_decision.json \
  --trading-file ./output/AAPL_trading.json \
  --risk-file ./output/AAPL_risk.json \
  --output ./output
```

### 快速生成投资计划

```bash
python scripts/quick_plan.py \
  --symbol AAPL \
  --risk-profile neutral \
  --output ./output
```
